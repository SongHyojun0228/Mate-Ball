# Claude Code 개발 가이드

## 개발 환경
```bash
npm install          # 의존성 설치
npm run dev          # 개발 서버 (http://localhost:5173)
npm run build        # 프로덕션 빌드
```

## 환경 변수 (.env)
```
VITE_SUPABASE_URL       # Supabase 프로젝트 URL
VITE_SUPABASE_ANON_KEY  # Supabase Anonymous Key
```

## 기술 스택
- **Frontend**: React (Vite), Tailwind CSS v4, Zustand, React Router v6, Lucide React
- **Backend**: Supabase (Auth, PostgreSQL, RLS)
- **배포**: Vercel (프론트) + Supabase (백엔드)

## 디자인 시스템 컬러
- `primary` (#F5F0E8) — 배경, 카드 (야구공 흰색)
- `accent` (#C0392B) — CTA, 강조 (실밥 빨강)
- `dark` (#1A1A2E) — 헤더, 텍스트 (다크 네이비)
- `stitch` (#8B1A1A) — hover, 보조 강조
- `grass` (#2D5A27) — 태그, 뱃지 (잔디)
- `dirt` (#C4A882) — 보조 배경, 구분선 (흙)

## 프로젝트 구조
```
src/
├── components/       # GameCard, Layout, ProtectedRoute, PredictionTab, MatchTab
├── pages/            # Home, Login, Signup, TeamSelect, GameRoom, NotFound
├── lib/supabase.js   # Supabase 클라이언트
├── stores/authStore.js # Zustand 인증 상태
supabase/
└── migration.sql     # DB 스키마 + 시드 데이터 (Supabase SQL Editor에서 실행)
```

## DB 설정
1. Supabase 대시보드 → SQL Editor
2. `supabase/migration.sql` 전체 내용 붙여넣기 → Run
3. 팀 10개 + 샘플 경기 5개 자동 생성됨

## 주요 제약사항
- 응원팀은 시즌당 1회 변경 가능
- 경기방은 해당 팀 팬만 입장 가능 (프론트 검증 + RLS)
- 공익 기간(~2027/12/04): 모든 기능 무료 운영

## 4. 경쟁 서비스와의 차별점

### 경쟁사 분석

| 서비스 | 주요 기능 | 한계 |
|---|---|---|
| LIVE스코어 | 실시간 스코어, AI 승부예측, 커뮤니티 | 직관 매칭 없음. 사람 간 교류 없음 |
| 스탯티즈 | 상세 야구 통계, 승부예측 | 데이터 중심. 감성/교류 없음 |
| 메잇볼 | 직관 메이트 매칭 | 승부예측/내기/추첨 없음. 재방문 이유 부족 |

### 야메의 차별점

1. **경기방 구조** — 해당 경기 팬만 입장. 기아팬 + 롯데팬이 같은 방에서 교류
2. **재방문 루프** — 승부예측 → 응모권 → 치킨 추첨으로 매일 들어올 이유
3. **직관 내기** — 상대팀 팬과 가벼운 내기. 야구 문화의 핵심 재미 구현
4. **직관 크루** — 1회성 매칭이 아닌 시즌 내내 함께하는 고정 모임
5. **평점 시스템** — 매너 기반 신뢰 매칭. 직관 후 서로 평점

## 6. 핵심 기능 상세

### 6-1. 경기방 (Game Room)

```
- KBO 일정 기반 자동 생성 / 경기 종료 후 자동 마감
- 입장 조건: 본인 응원팀이 해당 경기에 포함된 팬만 입장
- 예시: 기아 vs 롯데 → 기아팬 + 롯데팬만 입장 가능
- 응원팀 설정: 회원가입 시 1개 선택, 시즌당 1회 변경 가능
- 방 구성: 승부예측 탭 / 직관매칭 탭 / 응원토크 탭
```

### 6-2. 승부 예측 (Prediction)

```
- 경기 시작 전까지 참여 가능
- 예측 항목: 승/패/무 선택 + 한 줄 예측 이유 (선택)
- 경기 종료 후 자동 결과 반영 + 응모권 자동 지급
- 예측 이유는 상대팀 팬에게도 공개 → 자연스러운 설전 유도

[무료] 하루 2경기 예측, 응모권 1장
[구독] 하루 5경기 예측, 응모권 5장, 시즌 누적 적중률 통계
```

### 6-3. 직관 매칭 (Gwan Match)

```
- 같은 경기방 안에서 직관 동행 구하기
- 같은 팀 팬끼리 or 상대팀 팬끼리 모두 가능
- 동행 글 작성 항목:
  · 좌석 구역: 1루 / 3루 / 외야 / 미정
  · 선호 성별: 무관 / 동성만
  · 모집 인원: 1:1 / 단체
  · 한 줄 소개

[무료] 하루 2번 매칭 요청
[구독] 무제한 요청, 상대방 직관 히스토리 열람, 좌석/성별 필터
```

### 6-4. 직관 평점 (Rating)

```
- 직관 후 서로 매너 평점 (1~5점) 남기기
- 누적 평점으로 "직관 매너 뱃지" 부여
- 신뢰 기반 매칭 문화 형성

[무료] 평점 남기기 가능, 내 평점만 확인
[구독] 상대방 평점 열람 가능
```

### 6-5. 직관 크루 (Crew)

```
- 1회성 매칭이 아닌 시즌 내내 함께하는 고정 직관 모임
- 크루명, 응원팀, 소개 설정
- 크루 게시판에서 직관 일정 조율

[무료] 크루 참여만 가능 (최대 5명 크루)
[구독] 크루 개설 가능, 무제한 인원
```

### 6-6. 직관 내기 (Bet)

```
- 같은 경기방 상대팀 팬과 1:1 가벼운 내기
- 예시: "기아 이기면 내가 커피, 지면 네가 커피"
- 내기 신청 → 상대방 수락 → 경기 종료 후 결과 자동 공지
- 실제 정산은 앱 밖에서 당사자 간 진행 (금전 중개 없음)
- ⚠️ 법적 이슈 회피: 앱은 "약속 기록" 역할만, 직접 정산 없음

[무료] 내기 수락만 가능
[구독] 내기 신청 가능, 내기 히스토리 확인
```

### 6-7. 주간 치킨 추첨 (Chicken Draw)

```
- 매 라운드 마지막 경기 종료 직후 자동 추첨
- 당첨 인원: 매주 10명
- 상품: 치킨 1마리
- 재원:
  · 공익 기간(~2027/12): 운영자 송혜준 사비
  · 이후: 구독 수익 일부 + 스폰서

[무료] 응모권 1장 (예측 참여 시 자동 지급)
[구독] 응모권 5장
⚠️ 무료 유저도 참여 가능 → 사행성 이슈 최소화
```

## 11. 기술 스택

### Frontend
```
- Framework: React (Vite)
- Styling: Tailwind CSS
- State: Zustand
- Router: React Router v6
```

### Backend / DB
```
- Supabase
  · Auth: 이메일/소셜 로그인
  · Database: PostgreSQL
  · Realtime: 경기방 실시간 업데이트
  · Edge Functions: 경기 스크래핑, 추첨 자동화
  · Storage: 팀 로고, 유저 프로필 이미지
  · RLS (Row Level Security): 경기방 팬 입장 제한
```

### 배포
```
- Vercel (프론트엔드)
```
